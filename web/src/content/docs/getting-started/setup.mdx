## Command Installation (VSCode)

1. Ensure you have your project initialized with your chosen frameworks as well as [TailwindCSS](https://tailwindcss.com/docs/installation).

2. Follow the steps under the *init* command in the [VScode Extension](/docs/getting-started/vsc) section.

## Manual Installation

3.  ### Set up tailwind.config

    Add the preset to your tailwind config.

    <div class="max-h-72 overflow-y-auto no-scrollbar">
    ```js
    module.exports = {
      presets: [
        require('./tailwindPreset.js')
      ],
       // ...
    }
    
    ```
    </div>

    *If you are using next.js with the App dir, you will likely need to add `./potions/**/*.{ts,tsx}` to your tailwind content array.

## Optional Configurations

1. ### Path Alias

    NextJS will look something like
    ```json
    {
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "@/*": ["./*"]
        }
      }
    }
    ```
    
    Astro will look something like
    ```json
    {
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "@potions/*": ["src/potions/*"],
        }
      }
    }
    ```

2.  ### TW IntelliSense

    If you are using VScode and would like to enable auto-complete/highlights (for Tailwind Variants), add the following to your settings.json.


    ```json
    {
      "tailwindCSS.experimental.classRegex": [
         ["tv\\(([^)]*)\\)", "[\"'`]([^\"'`]*).*?[\"'`]"]
       ]
    }
    ```

3.  ### TW-Prettier class sorting

    If you are using the Tailwind-Prettier sorting plugin, and would like this plugin to sort classes within Tailwind Variants, add the following to your module.exports inside your Prettier.config.


    ```json
      tailwindFunctions: ["tv"],
    ```
